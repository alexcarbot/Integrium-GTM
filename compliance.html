Validation Flow

Four-step billing validation pipeline

Every wound care encounter passes through four deterministic validation gates before claim submission. Each step produces a RuleTrace record.

1

Calculate CPT

Wound assessment data maps to CPT codes through deterministic rules — debridement depth, wound size, and procedure type drive code selection.

2

Validate NCCI

Selected CPT codes are checked against 26 wound-care NCCI edit pairs. Bundling conflicts are caught with modifier bypass detection before submission.

3

Check LCD

Documentation is validated against 13 LCD rules for wound care. Missing medical necessity evidence or documentation gaps are flagged for correction.

4

Sign & Lock

Final gate validates ICD-10 specificity, laterality, diagnosis pointer order, E&M level, smoking status, and POA indicators before claim release.

FAQ

Compliance questions, answered

Integrium CORE validates 26 wound-care-specific NCCI edit pairs covering debridement, wound care management, negative pressure therapy, skin substitute grafts, and E&M combinations. The engine checks Column 1/Column 2 relationships with PTP indicator awareness, and detects when modifier bypass (59, XE, XS, XP, XU) is applicable to unbundle procedures.
The system validates wound care encounters against 13 LCD rules that check for required documentation elements — wound measurements, treatment rationale, medical necessity evidence, and clinical progress indicators. When documentation gaps are detected, the system flags specific missing elements before claim submission so clinicians can correct them in real time.
Sign & Lock is the final billing gate before claim release. It runs seven deterministic checks: ICD-10 code specificity, laterality requirements, diagnosis pointer order, E&M level appropriateness, smoking status documentation, POA (Present on Admission) indicators, and overall claim completeness. Every check produces a pass/fail result with a RuleTrace record explaining the decision.
Integrium CORE includes a CMS quarterly refresh pipeline that ingests updated NCCI edit tables, LCD revisions, and code set changes from authoritative CMS distributions. Rule updates are version-controlled and applied with effective dates, so the system always validates against the rules that were in effect at the time of service.
